BM25 FAQ 命中流程

用户问题先分词，再与标准问题计算 BM25 得分。Softmax 后最高分超过阈值就直接返回 FAQ 答案。

query -> preprocess -> scores -> softmax -> threshold

Milvus 混合检索

BGE-M3 同时生成稠密向量和稀疏向量，Milvus 混合检索后用 CrossEncoder 重排父文档。

dense + sparse -> WeightedRanker -> rerank

RAGAS 评估闭环

把问题、答案、上下文和参考答案送入评估指标，反向指导检索参数、分块策略和 Prompt 调整。

dataset -> answer -> metrics -> improve

Docker 部署链路

把应用代码、依赖安装和启动命令写入 Dockerfile，构建镜像后在服务器以容器方式运行。

Dockerfile -> image -> container -> service

术语速查

Redis

内存型键值数据库，本项目用于缓存标准问题、分词结果和已命中的答案。

BM25

关键词相关性算法，用于判断用户问题和 FAQ 标准问题是否足够相似。

Softmax

把原始分数转成概率分布，方便用最高分和阈值判断是否命中。

Milvus

向量数据库，用于保存知识库文档的 dense/sparse 向量并执行混合检索。

Rerank

对候选文档重新排序，把更适合生成答案的上下文排到前面。

BERT 意图识别

判断问题属于通用知识还是专业咨询，决定是否进入 RAG 检索流程。

RAGAS

RAG 系统评估工具，用指标检查答案忠实度、相关性、上下文精确率和召回率。

Docker

容器化部署工具，用镜像固化运行环境，让应用更容易迁移和上线。
