RAG 检索策略：从“直接查”到“拆开查、假设查、递进查”

这页把截图中的四类检索策略整理成适合课堂讲解的 HTML 网页，并对术语和知识点做了校正。核心逻辑是：问题越明确，检索越直接；问题越复杂、抽象或深层，越需要改写、拆分、假设文档或多轮递进检索。

整体结论：
截图中的四类思路大方向正确，可以用于讲 RAG 检索策略。
主要修正：
“回溯检索”不是常见标准叫法，建议改成“递进式检索 / 迭代检索 / 查询改写检索”。
讲课重点：
不是每个问题都用高级策略，应该根据问题复杂度选择最小有效检索方案。
四种策略

一、检索策略讲解

1直接检索

适合：明确查询、事实型问题、关键词比较清楚的问题。

大白话 最直接的“关键词找答案”：问题像“填空题”，有明确指向，就直接用问题里的关键词去搜资料。
核心逻辑 用户问题中已经包含明确实体、时间、指标或关键词，系统不需要复杂改写，直接把问题或关键词送入向量检索、关键词检索或混合检索。
例子 用户问：“公司 2025 年的研发投入占比是多少？”
  1. 提取关键词：公司、2025 年、研发投入占比。
  2. 到数据库或文档库中匹配相关段落。
  3. 返回能直接支撑答案的资料片段。
优点 快、成本低、链路短，适合简单明确的查询。

2子查询检索

适合：多条件、多维度、需要拆分后再综合的问题。

大白话 把一个大问题拆成几个小问题，每个小问题单独检索，最后把结果合并起来回答。
核心逻辑 用户问题包含多个子主题、多个维度或多个比较对象。先拆分成若干子查询，再分别检索，最后做证据去重、对齐和综合判断。
例子 用户问：“对比广州和深圳的人才引进政策，从补贴金额、落户难度、产业适配性三个方面分析哪个更适合计算机专业毕业生？”
  1. 拆成子问题：广州补贴、广州落户、广州产业；深圳补贴、深圳落户、深圳产业。
  2. 每个子问题分别检索。
  3. 把结果按“城市 × 维度”整理成对比结构。
  4. 根据证据给出综合结论。
优点 能避免复杂问题“一口吃不下”，让每个检索任务更聚焦，减少遗漏关键信息。

3HyDE 检索

适合：抽象问题、开放式问题、原始问题缺少明确关键词的问题。

大白话 先“假设性写一段可能的答案或文档”，再用这段内容去检索相似资料。它不是直接相信假设答案，而是把假设文本当作检索桥梁。
核心逻辑 HyDE 的更严谨说法是 Hypothetical Document Embeddings：让模型生成一段“假设文档”，把这段假设文档向量化，再用向量去找真实文档。
例子 用户问：“如何培养孩子的时间管理能力？”
  1. 先让模型生成假设文档：例如包含“作息表、计时器、任务拆分、家长监督”等内容。
  2. 把假设文档向量化。
  3. 用这个向量去检索真实资料。
  4. 用真实资料回答，而不是直接用假设答案回答。
优点 能补足原问题缺少关键词的问题，让检索更容易命中概念相关资料。

4递进式检索 （替代“回溯检索”叫法）

适合：深层问题、研究型问题、需要先找背景再找细节的问题。

大白话 先把复杂问题简化，找到基础资料；再根据基础资料继续追问细节，像“剥洋葱”一样一层层深入。
核心逻辑 对复杂问题先做查询改写或“退一步提问”，获得背景信息；再基于已检索到的内容生成下一轮更具体的查询，逐层补充证据。
例子 用户问：“元宇宙技术在教育领域的应用瓶颈及未来 3 年的突破方向？”
  1. 第一轮：简化为“元宇宙 教育应用 瓶颈”，找到基础信息。
  2. 第二轮：根据基础信息继续检索硬件成本、内容制作、交互体验、教师培训等细节。
  3. 第三轮：结合行业趋势，分析未来 3 年的突破方向。
优点 适合深层分析题，能够先搭框架，再逐步补证据，避免一开始就陷入细节检索混乱。
对比记忆

二、四种策略怎么选？

策略 最适合的问题 检索动作 一句话记忆
直接检索 问题明确，关键词、实体、时间、指标清楚 直接用原问题或关键词检索 能直接查，就别复杂化
子查询检索 一个问题包含多个对象、多个条件、多个维度 拆成多个子问题分别查，再合并 大问题拆小问题
HyDE 检索 问题抽象，缺少明确关键词，但语义方向存在 先生成假设文档，再用假设文档向量检索 先造检索桥梁，再找真实资料
递进式检索 深层研究题、趋势分析题、需要背景和细节的问题 先查背景，再根据结果继续查细节 先搭框架，再逐层深入
校对结果

三、截图知识点校对与修正建议

1. 直接检索：正确

截图中“适用于明确查询、直接用关键词搜资料”的说法是正确的。建议补充：实际系统中直接检索不一定只用关键词，也可以用向量检索或混合检索。

2. 子查询检索：基本正确

“分解复杂查询”的解释正确。需要补充的是：子查询结果不能简单拼接，最好做证据去重、来源对齐、冲突检查，再形成最终答案。

3. HyDE 检索：需要更严谨

截图里“先猜后搜”的大白话容易让学生误以为系统会直接相信假设答案。更准确的讲法是：生成假设文档作为语义检索桥梁，最终答案仍然要依据真实检索资料。

4. “回溯检索”：建议改名

“回溯检索”不是 RAG 里特别标准的通用叫法。截图描述的本质更像“查询改写 + 迭代检索 / 递进式检索 / Step-back 检索”。用于教学时建议用“递进式检索”更容易理解。

课堂总结

四、学生版记忆口诀

简单问题：直接检索。

复杂多维问题：子查询检索。

抽象开放问题：HyDE 检索。

深层研究问题：递进式检索。

最终原则：检索策略不是越复杂越好，而是越匹配问题越好。能直接查就直接查，查不准再考虑拆分、改写、HyDE 或多轮递进。
