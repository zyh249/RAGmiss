# BERT 分类报告与混淆矩阵

## 一、场景说明

在 BERT 二分类任务中，模型需要把用户问题分成两类：

```text
0 = 通用知识
1 = 专业咨询
```

你的分类报告如下：

```text
              precision    recall  f1-score   support

通用知识       0.89      1.00      0.94        25
专业咨询       1.00      0.96      0.98        72

accuracy                           0.97        97
macro avg       0.95      0.98      0.96        97
weighted avg    0.97      0.97      0.97        97
```

混淆矩阵如下：

```text
[[25  0]
 [ 3 69]]
```

这份报告整体效果很好，模型总准确率达到 97%。

---

## 二、support 是什么？

`support` 表示每个类别在验证集中真实存在多少条样本。

```text
通用知识：25 条
专业咨询：72 条
总计：97 条
```

所以当前验证集一共有 97 条数据。

其中：

```text
通用知识样本较少
专业咨询样本较多
```

---

## 三、混淆矩阵怎么看？

你的混淆矩阵是：

```text
[[25  0]
 [ 3 69]]
```

如果标签顺序是：

```python
["通用知识", "专业咨询"]
```

那么矩阵可以展开成：

| 真实类别 | 预测为通用知识 | 预测为专业咨询 |
|---|---:|---:|
| 通用知识 | 25 | 0 |
| 专业咨询 | 3 | 69 |

也就是说：

```text
真实是通用知识的 25 条：
全部预测成了通用知识，错 0 条。

真实是专业咨询的 72 条：
69 条预测成专业咨询，3 条误判成通用知识。
```

所以模型一共错了 3 条：

```text
错误数 = 3
正确数 = 25 + 69 = 94
总数 = 97
```

---

## 四、accuracy 准确率怎么看？

`accuracy` 表示总体预测正确率。

公式：

```text
accuracy = 预测正确的样本数 / 总样本数
```

代入你的结果：

```text
accuracy = (25 + 69) / 97
accuracy = 94 / 97
accuracy ≈ 0.969
```

四舍五入后：

```text
accuracy = 0.97
```

也就是：

```text
97 条验证数据中，有 94 条预测正确，3 条预测错误。
```

---

## 五、precision 精确率怎么看？

`precision` 关注的是：

```text
模型预测成某个类别的样本里，有多少是真的？
```

### 1. 通用知识 precision = 0.89

模型预测为“通用知识”的样本有：

```text
25 + 3 = 28 条
```

其中真正是“通用知识”的有：

```text
25 条
```

所以：

```text
通用知识 precision = 25 / 28
通用知识 precision ≈ 0.89
```

含义：

```text
模型只要预测某条问题是“通用知识”，有 89% 的概率是真的通用知识。
```

为什么不是 100%？

因为有 3 条“专业咨询”被误判成了“通用知识”。

---

### 2. 专业咨询 precision = 1.00

模型预测为“专业咨询”的样本有：

```text
69 条
```

这 69 条全部都是真的“专业咨询”。

所以：

```text
专业咨询 precision = 69 / 69
专业咨询 precision = 1.00
```

含义：

```text
模型只要预测某条问题是“专业咨询”，当前验证集上没有预测错。
```

---

## 六、recall 召回率怎么看？

`recall` 关注的是：

```text
真实属于某个类别的样本里，模型找回了多少？
```

### 1. 通用知识 recall = 1.00

真实“通用知识”有：

```text
25 条
```

模型正确识别了：

```text
25 条
```

所以：

```text
通用知识 recall = 25 / 25
通用知识 recall = 1.00
```

含义：

```text
所有真实的通用知识，模型都识别出来了。
```

---

### 2. 专业咨询 recall = 0.96

真实“专业咨询”有：

```text
72 条
```

模型正确识别了：

```text
69 条
```

漏掉了：

```text
3 条
```

所以：

```text
专业咨询 recall = 69 / 72
专业咨询 recall ≈ 0.96
```

含义：

```text
真实的专业咨询里，有 96% 被模型成功识别出来。
剩下约 4% 被误判成了通用知识。
```

---

## 七、f1-score 怎么看？

`f1-score` 是 precision 和 recall 的综合指标。

可以简单理解为：

```text
f1-score = precision 和 recall 的综合平均分
```

你的结果是：

```text
通用知识 f1-score = 0.94
专业咨询 f1-score = 0.98
```

说明：

```text
专业咨询识别效果更好。
通用知识整体也不错，但 precision 稍低。
```

通用知识 precision 较低的原因是：

```text
有 3 条专业咨询被误判成了通用知识。
```

---

## 八、macro avg 和 weighted avg 怎么看？

### 1. macro avg

`macro avg` 表示每个类别指标的普通平均。

它不考虑每个类别样本数量多少。

例如 precision：

```text
通用知识 precision = 0.89
专业咨询 precision = 1.00

macro precision = (0.89 + 1.00) / 2
macro precision = 0.945
macro precision ≈ 0.95
```

所以 `macro avg` 更关注：

```text
每个类别是否都表现好。
```

如果某个小类别效果很差，`macro avg` 会明显下降。

---

### 2. weighted avg

`weighted avg` 表示按每个类别的样本数量进行加权平均。

你的验证集中：

```text
通用知识：25 条
专业咨询：72 条
```

因为“专业咨询”样本更多，所以 `weighted avg` 更受“专业咨询”影响。

因此你的 `weighted avg` 是：

```text
weighted avg = 0.97
```

它更接近样本数量多的类别表现。

---

## 九、precision 和 recall 的区别

### precision 关注预测结果准不准

问题是：

```text
模型说它是这个类别，它到底是不是真的？
```

例如：

```text
模型预测为“通用知识”的 28 条里，只有 25 条是真的。
所以通用知识 precision = 25 / 28 = 0.89。
```

---

### recall 关注真实样本有没有被找回来

问题是：

```text
真实属于这个类别的样本，模型有没有全部找出来？
```

例如：

```text
真实专业咨询有 72 条，模型找回 69 条。
所以专业咨询 recall = 69 / 72 = 0.96。
```

---

## 十、对 RAG 系统来说，最应该看哪个指标？

你的分类器大概率用于 RAG 系统前置分流：

```text
通用知识 -> 直接由大模型回答
专业咨询 -> 走 RAG 检索 / 知识库问答
```

在这个业务场景中，最危险的错误是：

```text
真实是专业咨询，却被预测成通用知识。
```

因为这样会导致：

```text
本来应该走知识库检索的问题，没有进入 RAG 流程。
大模型可能直接回答，容易出现不准确或编造。
```

在你的混淆矩阵中：

```text
[[25  0]
 [ 3 69]]
```

最需要关注的是这一项：

```text
真实专业咨询 -> 预测通用知识 = 3 条
```

所以对 RAG 分类器来说，重点指标是：

```text
专业咨询 recall
```

你当前：

```text
专业咨询 recall = 0.96
```

已经不错。

如果企业项目中专业咨询问题非常重要，可以继续优化到：

```text
专业咨询 recall >= 0.98
```

## 十一、这份报告的结论

可以总结为：

```text
模型整体表现较好，验证集准确率达到 97%。

通用知识类别 recall 为 1.00，说明所有真实通用知识都被正确识别。

专业咨询类别 precision 为 1.00，说明模型只要判断为专业咨询，当前验证集上没有误判。

当前主要问题是有 3 条专业咨询被误判为通用知识，导致专业咨询 recall 为 0.96，同时通用知识 precision 降到 0.89。
```

## 十二、下一步应该怎么优化？

最应该做的是打印这 3 条错分样本。

示例代码：

```python
for text, true_label, pred_label in zip(val_texts, val_labels, predictions):
    if true_label != pred_label:
        print("问题：", text)
        print("真实标签：", true_label)
        print("预测标签：", pred_label)
        print("-" * 50)
```

然后分析：

```text
1. 这 3 条问题是不是标注错了？
2. 这 3 条问题是不是太短？
3. 这 3 条问题是不是专业词不明显？
4. 训练集中是否缺少类似表达？
5. 是否需要补充更多“容易被误判”的专业咨询样本？
```

如果确认标注没问题，就把类似问题补充到训练集中。
